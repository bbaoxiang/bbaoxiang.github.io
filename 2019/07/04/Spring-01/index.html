txt dat word excel …..

库
创建库
create database db1;—sql语句

create database db2 character set utf8;–sql语句以 ; 结尾

创建表的同时设置字符集和引擎

create table +t2(name varchar(10),age int)engine = myisam charset = gbk;

engine:引擎

引擎：innodb：支持数据库的高级操作，如 外键 事物等
myisam：只支持基本的增删改查操作

查看表字段：desc 表名;

删除表 drop table 表名;

修改表的引擎和字符集

字符集：一张表（Ascii表）

utf8:万国码 中文2个字节，英文1个字节

gbk:国标 中文3个字节 英文1个字节

iso8859-1:欧标，不支持中文

alter table engine = innodb/myisam charset = utf8
修改表名：rename table 表名 to 新名
修改表的字段
after table 表名 add 字段名 字段类型 after ***(first)

增删改查

插入数据：

指定字段插入:insert into 表名(字段1，字段2,…) values(字段1数据,字段2数据,….);

批量数据插入：insert into 表名(字段1，字段2,…) values(字段1数据,字段2数据,….),(字段1数据,字段2数据,….),(字段1数据,字段2数据,….);

约束：主键约束

主键：用于表示数据的唯一性，非空性

约束：给字段添加的限制条件

create table t1(id int primary key,name varchar(20));

insert into t1 value(1,'zs');

insert into t1 value(null,'zs');

insert into t1 value(2,'ls');

insert into t1 value(3,'ww');

insert into t1 value(1,null);

自增：auto_increment

create table t2(id int primary key auto_increment,name varchar(20));

insert into t2 value(null,'zs'),(null,'ls');

delete from t2 where id=2;

insert into t2 value(null,'ww');

注意：自增的第一个值是1，删除数据，不会删除主键自增值

表设计案例：权限管理

实现权限管理功能需要三张主表

create table user(id int,name varchar(10));   1

create table role(id int,name varchar(10));   2

create table module(id int,name varchar(10));  3

两张关系表：

create table ur(uid int,rid int);             4

create table rm(rid int,mid int);             5

insert into user values(1,'刘德华'),(2,'罗玉凤');

insert into role values(1,'男游客'),(2,'男会员'),(3,'女游客'),(4,'女管理员');

insert into module values(1,’男浏览’),(2,’男发帖’),(3,’女浏览’),(4,’女发帖’),(5,’女删帖’);

insert into ur values(1,2),(1,3),(2,1),(2,4);

insert into rm values(1,1),(2,2),(2,1),(3,3),(4,3),(4,4),(4,5);

1.查看每个用户的权限有哪些？

select u.name,m.name
from user u join ur ur on u.id = ur.uid
          join rm rm on rm.rid = ur.rid
          join module m on m.id = rm.mid;

2.查看凤姐的权限有哪些？

select u.name,m.name
from user u join ur ur on u.id = ur.uid
          join rm rm on rm.rid = ur. 
          join module m on m.id = rm.mid
          where u.name='罗玉凤';

3.查看拥有男浏览权限的用户是谁？

select u.name,m.name
from user u join ur ur on u.id = ur.uid
          join rm rm on rm.rid = ur.rid
          join module m on m.id = rm.mid
          where m.name='男浏览';

Spring

什么是框架？
框架本身就是一个半成本软件，使用框架可以简化应用软件的开发，提高开发效率。

什么是Spring框架？

由Spring开源社区维护的“系列”框架

核心组件：IOC / AOP

IOC:控制反转，创建对象的功能交给容器来完成。

Spring bean对象作用域（scope）

Spring容器默认情况下按照“单例”管理对象—>单例模式

单例：在软件中对象是唯一的一个实例现象
多次调用getBean获得的是同一个对象的引用

多例：Spring使用bean标签的scope属性可以指定多例

bean对象生命周期的方法

init-method用于设定初始化方法，创建对象之后立即执行

destroy-method用于设定销毁方法，关闭容器销毁对象时候执行

仅对单例模式有效，多例对象无效

懒惰初始化

Spring容器默认情况下单例对象是立即初始化的，也就是WSpring容器在启动的时候立即创建单例对象实例

lazy-init:延迟初始化，也就是在第一次调用getBean方法时候才会创建对象的实例

*:在开发中少用，消耗的内存太大
